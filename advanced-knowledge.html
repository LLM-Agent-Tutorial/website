Injecting Domain Knowledge

Injecting domain knowledge steers a general-purpose LLM into an expert agent. The tutorial highlights a progression from lightweight prompt injections to retrieval pipelines and parameter-efficient fine-tuning that permanently encode expertise.

Domain knowledge playbook

Use all three levers together: start with examples and glossaries in the prompt, layer on retrieval from authoritative corpora, and only then update model weights when behavior needs to be baked in.

  • In-context learning: Place relevant facts, rubrics, or workflows directly in the prompt so the agent reasons with on-demand briefing. A simple triage prompt—“Use the following medical knowledge... chest pain with exertional onset... requires immediate ER evaluation.”—already enforces expert heuristics.
  • Retrieval-Augmented Generation: Pull only the highest-signal snippets from textbooks, knowledge graphs, database queries, or live search, then stitch them into the prompt so each response is grounded in fresh evidence.
  • Fine-tuning / PEFT: Embed domain knowledge inside the weights with LoRA or adapters. Once tuned, the agent behaves consistently even when prompts are short, because the expertise now lives in the model parameters.
