Mobile Agent: Operate the Mobile Phone as Smoothly as a Human

Paper ↗ GitHub ↗

The Mobile Agent assists users by simplifying interactions with mobile devices, making complex operations easier and more intuitive. Users provide high-level instructions in natural language, and the agent interprets those instructions, identifies the necessary steps, and automatically performs actions such as tapping buttons, entering text, or navigating across apps, significantly streamlining task completion.

Planning

The agent interprets the user's instruction in the context of the device's current state (typically represented by the latest screenshot). Using this understanding, it formulates an operational plan and identifies the optimal next action, translating high-level goals into precise swipes, taps, or text entry.

Example of planning the next action

INPUT: Instruction: Set a clock for 7:00 a.m. Screenshot: ... History: {...} Task: Think and select the next action (swipe, tap, type ...).

OUTPUT: Operation Thought: Currently on the home screen we need to first tap the clock icon ... Action: Tap(500,800).

Tool use

After generating the next action, the agent directly inputs it into an action mapping tool, which converts the natural-language action into executable instructions for the device. Tap(500,800) becomes adb shell input tap 500 800, while swipes, text entry, and app launches are translated into the appropriate Android Debug Bridge commands, connecting high-level planning with concrete device operations.

Reflection

After each operation, the agent evaluates its effectiveness by comparing the screenshot before and after the action. If the outcome diverges—such as when an incorrect page opens—the agent detects the error and initiates corrective measures so failures do not cascade.

Example of reflection for checking operation success

INPUT: Last Screenshot: ... Current Screenshot: ... Latest Operation: ... Task: Compare and determine whether the latest operation was successful.

OUTPUT: "By comparing these two images, it is found ... so this operation is successful."

Memory

Throughout the workflow, the agent maintains a memory of task-relevant information that may be required in later steps. It records intermediate results and contextual details across applications, ensuring continuity for multi-step and cross-application tasks.
