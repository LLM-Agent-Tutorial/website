3.8 Overview

Operate the mobile phone as smoothly as a human

The Mobile Agent assists users by translating natural language requests into precise swipe, tap, and type sequences. Users provide high-level objectives; the agent interprets the current screenshot and history, devises a plan, and executes the necessary gestures, minimizing manual effort.

Planning

Reasoning about the next action

The agent combines the user's instruction with the live screenshot to decide what to do next. Example prompt:

Example · Planning next touch

Instruction: Set a clock for 7:00 a.m. Screenshot: [home screen]. Task: Think and select the next action.

Output: Operation Thought: Currently on the home screen we need to first tap the clock icon... Action: Tap(500,800).

Tool use

Mapping natural language to ADB commands

After planning, the action is sent to an action-mapping tool that converts the abstract gesture into executable Android Debug Bridge commands. Tap(500,800) becomes adb shell input tap 500 800. Swipes, text entry, and app launches are handled the same way, closing the loop between LLM reasoning and device control.

Reflection & memory

Comparing screenshots to verify success

After each gesture the agent compares before/after screenshots. If the outcome diverges — for example an incorrect page opens — the agent detects the failure, revises the plan, and retries so that errors do not cascade. Throughout the workflow it also stores relevant context (intermediate results, app state) for later steps.

Example · Verifying the last action

"By comparing these two images, it is found ... so this operation is successful."

When the screenshots disagree, the agent flags failure and replans immediately.
