Managing Long Context

Long contexts raise cost and can blur attention. The tutorial advocates compressing history, reusing stable prefixes, and introducing hierarchy so agents never juggle more information than needed.

Context management toolkit

Choose the tactic that matches your bottleneck: aggressively compress stale content, cache reusable prompt segments, and break workflows into hierarchical stages so each sub-agent sees only what it needs.

  • Compression: Summarize transcripts, chunk memory into retrieval stores, or offload rarely used facts to external notes so that only the salient snippets stay in-context.
  • Reuse: Prompt caching, byte-identical prefixes, and masked tool definitions preserve context without re-sending everything—crucial when staying within strict token budgets.
  • Hierarchy: Delegate subtasks to specialized agents with concise prompts and require structured summaries between them. Hierarchy keeps any single context window short while retaining global coherence.
