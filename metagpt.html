A multi-agent framework that writes code efficiently

MetaGPT mirrors established software engineering practices by assigning specialized roles, structuring plans, leveraging tools for automated coding, and iterating through inter-agent cooperation. Product Managers craft PRDs, Architects design systems, Engineers implement, Project Managers coordinate, and QA ensures quality.

Role-play

Product Manager

Name: Alice. Goal: create PRDs or market research while using the user's language.

Architect

Receives requirements via the message pool and drafts system designs with modules, interfaces, and files.

Engineer & QA

Engineers implement features using editor/terminal tools; QA validates requirements and test plans.

Planning

Planning is initiated by the Product Manager. Given "Make the 2048 sliding tile number puzzle game," MetaGPT outputs product goals, user stories, competitive analysis, and a prioritized requirement pool (core gameplay high, score tracking medium, responsive design low). This structured PRD anchors all downstream work.

Cooperation & tools

Cooperation revolves around a publish-subscribe message pool. Example message:

Example · Publish-subscribe message

{
  "content": "Bob (Architect), the PRD is ready at docs/PRD.md. Please draft the system design accordingly.",
  "send_to": "Bob"
}

When Engineers work, they choose from role-aligned tools. Before coding game.py, Alex issues Editor.open_file with path docs/system_design.json to inspect specs, demonstrating explicit reasoning behind each tool call.

Example · Tool invocation rationale

Reason: Before implementing game.py, I need to carefully review the complete system design.

Tool: Editor.open_file with parameters { "path": "docs/system_design.json" }.
