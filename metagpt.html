MetaGPT: Multi-Agent Framework that Write Code Efficiently

Paper ↗ GitHub ↗

MetaGPT is an LLM-based multi-agent framework designed to mirror established software engineering practices. It explicitly assigns specialized roles, structures detailed planning and task decomposition, leverages tools for automated code generation, and facilitates iterative debugging through inter-agent cooperation.

Role-play

Role-play serves as a foundational capability within MetaGPT, shaping how agents function and collaborate. Roles include Product Manager, Architect, Project Manager, Engineer, and QA Engineer. Role prompts define name, profile, goal, and constraints that govern responsibilities and interaction standards.

Example of role-playing as a product manager

Name: Alice

Profile: Product Manager

Goal: Create a Product Requirement Document or market research/competitive product research.

Constraint: Utilize the same language as the user requirements for seamless communication.

Planning

Planning involves systematically decomposing complex software engineering tasks into structured, actionable procedures clearly assigned to specific agent roles. The Product Manager analyzes user requirements and produces a detailed Product Requirements Document (PRD) that defines goals, user stories, competitive analysis, and a prioritized requirement pool, guiding downstream agents.

Example of planning for a product requirement document

INPUT: Agent description, high-level user requirements ("Make the 2048 sliding tile number puzzle game"), and the task to generate a structured PRD.

OUTPUT: Product Goals: Simple, responsive, user-friendly 2048 puzzle game for desktop and mobile. User Stories: Quickly start games, slide tiles to merge numbers, view scores, reset easily. Competitive Analysis: Original 2048 is simple but limited; mobile versions have good UI but intrusive ads; Threes! is polished yet paid. Prioritized Requirement Pool: Core gameplay (high), score tracking and reset (medium), responsive design and customization (low).

Cooperation

Cooperation revolves around the shared message pool mechanism. Agents publish structured outputs to the pool, other agents subscribe to relevant messages, and tasks flow through publish-subscribe interactions that preserve division of labor.

Example of the product manager assigning tasks to the architect

INPUT: Instruction to publish a message with task details, paths, requirements, and the receiving agent.

OUTPUT:

{
  "content": "Bob (Architect), the PRD is ready at docs/PRD.md. Please draft the system design accordingly.",
  "send_to": "Bob"
}

Tool use

Tool use is essential in MetaGPT. Product Managers employ market analysis utilities, Architects use design tools, and Engineers leverage coding tools (Editor, Terminal, Code Review) to streamline work. Engineers justify each tool call, specify parameters, and execute the associated shell commands.

Example of the engineer reviewing the system design using Editor.open_file

INPUT: Agent description for Alex (Engineer), system design from the Architect, current task to implement game.py, and access to Editor/Terminal/CodeReview tools.

OUTPUT: Reason: Before implementing game.py, I need to carefully review the complete system design. Tool: Editor.open_file. Tool Parameters: { "path": "docs/system_design.json" }.
