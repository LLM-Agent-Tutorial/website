A clear, grounded agent pattern

Embodied agents must translate language into safe behavior amid partial observability and uncertainty. PaLM-SayCan separates high-level reasoning (Say) from low-level feasibility (Can). At each step it asks: What is useful to do next, and what can the robot actually do now? Only actions that are both useful and feasible execute.

Planning (Say)

The LLM ranks named skills by usefulness.

Example · Planning probabilities

Instruction: I spilled my drink. Skills: [find sponge, pick up sponge, find apple].

Utilities: find sponge 0.85, pick up sponge 0.10, find apple 0.05.

Rationale: Locate a cleaning tool before manipulation.

Skill schema

Each skill exposes a schema with name, description, preconditions, postconditions, executor, and affordance model.

Example · Skill schema

Name: open_refrigerator
Description: Open the refrigerator door using the handle.
Preconditions: handle visible; gripper free
Postconditions: door angle > 20°
Executor: grasp → pull → release
Affordance: A_open_refrigerator(x) predicts success from detections, depth, reachability.

Reflection (Can)

For each skill, a pre-trained affordance function estimates likelihood of success from the current scene. Decision-making multiplies the Say utility with the Can feasibility.

Example · Combining Say & Can

Instruction: I'd like a cold apple. Scene: robot far from a closed refrigerator.

Scores: navigate (Say 0.50 × Can 0.95), open (0.40 × 0.20), pick up (0.10 × 0.05).

Action: navigate to refrigerator, after which opening becomes feasible.
