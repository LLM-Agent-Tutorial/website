3.7 Overview

Exploring, learning, and adapting autonomously in Minecraft

Voyager enables lifelong learning in the Minecraft sandbox. It generates its own curriculum of tasks, solves them with Mineflayer code, and refines behavior via iterative feedback and self-verification, building a growing library of reusable, compositional skills.

Planning

Automatic curriculum generation

Voyager's automatic curriculum module sets tasks based on current abilities and environment state, then decomposes them into actionable subtasks. This scaffolds the agent from simple to complex goals.

Example · Auto-generated task

Reasoning: The inventory is empty now, chop down a tree to get some wood.

Task: Obtain a wood log.

Tool use

Mineflayer programs for every action

Voyager controls the character via LLM-generated JavaScript built on the Mineflayer API. Each function is self-contained so it can be reused and composed:

Example · Generated Mineflayer skill

async function mineCobblestone (bot) {
  let woodenPickaxe = bot.inventory.findInventoryItem(...);
  if (!woodenPickaxe) {
    bot.chat("Crafting a wooden pickaxe.");
    await craftWoodenPickaxe(bot);
  }
  ...
}

Memory & reflection

Skill libraries plus self-verification

The skill library stores every verified program inside a vector database. When facing a new task, Voyager queries the library using embeddings of the task description and environment feedback to retrieve relevant skills for reuse and composition.

Reflection is powered by iterative prompting. After each attempt, Voyager checks success and issues critiques until the task is solved.

Example · Self-verification

You have enough materials to craft a wooden pickaxe, but you didn't craft it.

Success: False · Critique: Craft a wooden pickaxe with a crafting table using 3 spruce planks and 2 sticks.
