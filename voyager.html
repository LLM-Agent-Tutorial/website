Voyager: Exploring, Learning, and Adapting Autonomously in Minecraft

Paper ↗ GitHub ↗

Voyager is designed to enable lifelong learning in Minecraft. It generates its own curriculum of tasks, completes them through code-based actions, and refines its behavior via iterative feedback and self-verification, building a growing library of reusable skills that support progressively more sophisticated behaviors.

Below we examine how Voyager’s automatic planning, Mineflayer-based tool use, skill-memory retrieval, and reflective self-verification enable continual progress.

Planning

Voyager uses the automatic curriculum module for planning. The module autonomously generates new tasks based on current abilities and environment state, then decomposes high-level goals into actionable sub-tasks. When generating skill code, Voyager also produces the corresponding plan.

Example of an automatically generated task

INPUT: Instruction describing the mentor role, full game state (biome, time, inventory, entities, etc.), and a log of completed/failed tasks.

OUTPUT: Reasoning: "The inventory is empty now, chop down a tree to get some wood." Task: Obtain a wood log.

Tool use

Voyager controls its character by asking the LLM to write Mineflayer JavaScript. Mineflayer provides high-level APIs for moving, mining, crafting, and fighting. Each generated function is stored as an independent skill that can be retrieved later and composed into more complex behaviors.

Example of generated Mineflayer code

INPUT: Instruction to write Mineflayer code, useful programs, current game state, task ("Obtain a wood log"), context, and critique.

OUTPUT:

async function mineCobblestone(bot) {
  let woodenPickaxe = bot.inventory.findInventoryItem(
    mcData.itemsByName["wooden_pickaxe"].id
  );
  if (!woodenPickaxe) {
    bot.chat("Crafting a wooden pickaxe.");
    await craftWoodenPickaxe(bot);
    woodenPickaxe = bot.inventory.findInventoryItem(
      mcData.itemsByName["wooden_pickaxe"].id
    );
  }
  ...
}

Memory

Voyager exhibits memory through its skill library, an extensible vector database storing all verified executable programs. When a new task is proposed, Voyager queries the library using the task description and environment feedback to retrieve relevant skills, enabling composition of complex behaviors.

Reflection

An iterative prompting mechanism endows Voyager with reflection ability. After executing each generated program it observes environmental feedback, execution errors, and self-verification signals, revising the action code until the task succeeds.

Example of self-verification

INPUT: Instruction to evaluate task progress, current game state (inventory contains crafting table, spruce planks, sticks), and the task "Craft a wooden pickaxe."

OUTPUT: Reasoning: You have enough materials to craft a wooden pickaxe, but you didn't craft it. Success: False. Critique: Craft a wooden pickaxe with a crafting table using 3 spruce planks and 2 sticks.
